# Jamal OB (sweep-driven two-line) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build `jamal-ob.pine` — a standalone Pine v6 overlay that draws exactly two stepline "kill lines" (lower = bullish demand `open_R`, upper = bearish supply `open_R`), where each level comes from order-block sweep + walk-back logic and moves on every new same-side sweep.

**Architecture:** Regime-less, two fully independent mirrored sides. A "sweep" (red making a new low / green making a new high) triggers a backward walk-back over the contiguous leg; `R` = the highest-open red (bullish) / lowest-open green (bearish) in the leg; the line is set to `open_R` and held as a stepline until the next same-side sweep. No confirmation, gates, invalidation, boxes, or FSM in v1 (all deferred — see spec §7).

**Tech Stack:** Pine Script v6; developed and verified through the **TradingView MCP** (`pine_set_source` → `pine_smart_compile` → `data_get_study_values` / `replay_*` / `capture_screenshot`) driving the live TradingView Desktop chart. No local test runner — "tests" are chart reads against the validated NEAR-daily oracle from the parked spec.

**Spec:** `docs/superpowers/specs/2026-06-26-jamal-ob-design.md` (locked rules inherited verbatim from `docs/superpowers/specs/2026-06-09-jamal-ob-parked-state.md`).

- [ ] **Step 4: Add the walk-back function and use it**

Add this function in the functions area (after the Inputs block):

```pine
// ════════════════ Bullish walk-back (lower line = open_R) ════════════════
// From sweep bar T (offset 0, a red new-low), walk backward over the
// contiguous down-leg and return the highest red open (open_R). Termination
// is STRUCTURE-ONLY (green-that-held / doji). A mid-leg green pause — the bar
// toward T made a new low AND this green didn't break out up — is skipped.
// The running max selects R; it never stops the walk.
f_walkback_bull(int maxlb) =>
    float best_open = open                          // T is red — seed with its open
    for i = 1 to maxlb
        if close[i] < open[i]                       // red → in leg
            best_open := math.max(best_open, open[i])
            continue
        if close[i] > open[i]                       // green → pause or stop
            if low[i - 1] < low[i] and high[i] <= high[i + 1]
                continue                            // mid-leg pause: skip
        break                                       // green-that-held or doji → stop
    best_open
```

Then change the TEMP assignment from Step 1 to use it:

```pine
if barstate.isconfirmed and bull_sweep
    lower_line := f_walkback_bull(max_lookback)
```

### Task 3: Bearish side — mirror → upper line = `open_R`

**Files:**
- Modify: `jamal-ob.pine`

- [ ] **Step 1: Add the bearish walk-back function (mirror)**

Add after `f_walkback_bull`:

```pine
// ════════════════ Bearish walk-back (upper line = open_R) ════════════════
// Mirror of bullish: from sweep bar T (a green new-high), walk back the
// contiguous up-leg and return the lowest green open (open_R). Mid-leg red
// pause — bar toward T made a new high AND this red didn't break down — is
// skipped; red-that-held / doji stops the walk.
f_walkback_bear(int maxlb) =>
    float best_open = open                          // T is green — seed with its open
    for i = 1 to maxlb
        if close[i] > open[i]                       // green → in leg
            best_open := math.min(best_open, open[i])
            continue
        if close[i] < open[i]                       // red → pause or stop
            if high[i - 1] > high[i] and low[i] >= low[i + 1]
                continue                            // mid-leg pause: skip
        break                                       // red-that-held or doji → stop
    best_open
```
